设计目标

首页负责快速分流：读文章的人进入 Articles，想玩实验的人进入 Toys。 文章页则负责慢下来，让推导、图表、旁注和交互实验自然地放在同一条叙事里。

这个页面不依赖构建工具。每篇文章都是一个独立目录，可以有自己的 index.html、 script.js、图片、数据文件、WebGPU 或 WebGL 代码。GitHub Pages 会直接托管这些静态资源。

后续如果要写数学文章，可以在页面里接入 MathJax 或 KaTeX；如果要写机制解释类文章，可以直接把小实验写成本页脚本。

如何扩展

写新文章时，复制这个目录即可。文章自己的脚本放在同目录的 script.js， 样式如果很通用就放进 articles/article.css，如果只服务于单篇文章就放在文章目录里。 

articles/
  my-new-note/
    index.html
    script.js
    data.json

这个布局在桌面端保留左右栏：左侧目录，右侧旁注；移动端会自动变成单栏。

文章内 JS 推荐局部化，避免污染全局状态，后面文章多了也更好维护。
